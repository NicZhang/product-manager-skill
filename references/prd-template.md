# PRD 输出模板

本文件定义了 product-manager skill 的标准 PRD 输出格式。AI 在生成 PRD 时，必须严格按照以下模板结构输出，使用 Markdown 格式。

---

## 填充规则

在填写以下模板时，必须遵守以下规则：

1. **信息充分 → 完整填写**：凡是用户已提供的信息，必须完整、准确地填入对应位置，不得遗漏。
2. **信息不足 → 在第7节标注，不编造**：凡是用户未提供的信息，不得自行编造或凭空假设。应在对应位置填写 `[待补充]`，并在「第7节：开放问题与风险」中新增一条记录，说明缺失信息及其影响。
3. **验收标准可推导**：如功能描述清晰，可基于功能描述推导验收标准，但必须在每条推导的验收标准后标注 `[AI推导，待确认]`。
4. **优先级必须填写**：功能列表中的优先级字段（P0/P1/P2）必须填写，不可留空。
5. **参考资料统一归集**：用户提供的所有文件、截图、URL、竞品链接等，必须在「第8节：参考资料」中逐条记录，正文中可引用序号。
6. **表格格式统一**：所有表格必须使用标准 Markdown 表格语法，列宽对齐，内容简洁。
7. **版本号规则**：初稿固定为 `v0.1`，后续修订由人工维护。
